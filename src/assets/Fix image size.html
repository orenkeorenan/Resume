ChatGPT said:

Ah, I see exactly what's happening. The problem is that on tablet and mobile, your <img> is inheriting the full width of the container (width: "inherit"), but your container is already 100vw wide on smaller screens. That makes the image too big. Also, you haven’t restricted the image height, so it can overflow.

Here’s a better approach using responsive sizing with maxWidth and height: auto so it scales properly:

javascript
import { useScreenSize } from '../../../hooks/useScreenSize';
import PhotoProfile from '../../../assets/ProfilePhoto.jpg'

const PicturePanel = () => {
    const screen = useScreenSize();
    
    return (
        <div
            style={{
                width: screen === "desktop" ? "40vw" : "100vw",
                height: screen === "desktop" ? "100vh" : "30vh",
                position: screen === "desktop" ? "fixed" : "relative",
                backgroundColor: "blue",
                display: "flex",
                justifyContent: "center",
                alignItems: "center",
                overflow: "hidden", // ensures the image doesn't spill out
            }}
        >
            <img 
                src={PhotoProfile}
                style={{
                    width: "100%",    // fills container width
                    height: "auto",   // keeps aspect ratio
                    maxHeight: "100%",// prevents overflow in height
                    objectFit: "cover" // crops nicely if needed
                }}
            />
        </div>
    );
};

export default PicturePanel;

✅ Changes and why:

  1. width: "100%" and height: "auto": makes the image scale with container width but keep aspect ratio.

  2. maxHeight: "100%": ensures it never exceeds container height.

  3. objectFit: "cover": crops if the container ratio is different, so it looks neat.

  4. Added display: flex + justifyContent/alignItems: centers the image nicely.
